# Прогнозирование стоимости аренды: кросс-валидация, отбор признаков и оптимизация гиперпараметров

## 📋 Обзор

Это комплексное исследование в области машинного обучения, посвящённое прогнозированию цен на аренду квартир. В проекте реализованы и сравниваются:

- **5 собственных методов кросс-валидации** (KFold, стратифицированная, группированная, временные ряды) в сравнении с реализациями scikit-learn
- **3 метода отбора признаков** (Lasso, Permutation Importance, SHAP)
- **2 подхода к оптимизации гиперпараметров** (Grid Search против байесовской оптимизации с Optuna)

Проект демонстрирует, как различные стратегии валидации влияют на качество модели и какие признаки наиболее сильно влияют на стоимость аренды.

## 🔧 Предобработка данных

- Удаление выбросов с использованием 1-го и 99-го перцентилей по цене
- Создание бинарных признаков из 20 наиболее часто встречающихся удобств (например, лифт, консьерж, посудомоечная машина)
- Набор признаков: 23 признака (ванные комнаты, спальни, уровень интереса + 20 удобств)
- Целевая переменная: `price` (стоимость аренды в долларах США)

## 🧪 Реализованные методы

### 1. Методы разделения данных

| Функция | Описание |
|----------|-------------|
| `split_test_train()` | Случайное разделение (train/test) |
| `split_data()` | Случайное разделение (train/val/test) |
| `split_train_test_date()` | Разделение по времени (train/test) |
| `split_by_date()` | Разделение по времени (train/val/test) |

### 2. Методы кросс-валидации (собственные vs sklearn)

| Собственный метод | Эквивалент sklearn | Ключевая особенность |
|-------------------|-------------------|-------------|
| `my_KFold()` | `KFold` | Контроль перемешивания |
| `grouped_KFold()` | `GroupKFold` | Группировка по ванным/спальням |
| `stratified_Kfold()` | `StratifiedKFold` | Стратификация по квантилям цены |
| `time_KFold()` | `TimeSeriesSplit` | Учёт временной очерёдности |

### 3. Методы отбора признаков

| Метод | Тип | Описание |
|--------|------|-------------|
| **Lasso (L1)** | Встроенный | Обнуляет незначимые признаки |
| **Permutation Importance** | Модельно-агностический | Измеряет падение качества при перемешивании |
| **SHAP** | Теоретико-игровой | Объясняет индивидуальные предсказания |

### 4. Оптимизация гиперпараметров

- **GridSearchCV**: Полный перебор по 36 комбинациям
- **Optuna**: Байесовская оптимизация с 20 испытаниями (с K-Fold и без)

## 💡 Ключевые выводы

1. **Стратифицированная кросс-валидация** критически важна для задач прогнозирования цен, чтобы сохранить распределение цен по фолдам.

2. **Lasso** работает так же хорошо, как и более сложные методы (SHAP, Permutation) для отбора признаков на этом наборе данных, будучи при этом вычислительно более дешёвым.

3. **Главные предикторы цены**:
   - Количество ванных комнат (самый сильный)
   - Количество спален
   - Уровень интереса (высокий/средний/низкий)
   - Удобства в здании (консьерж, лифт, прачечная)

4. **Байесовская оптимизация** (Optuna) достигает качества Grid Search при значительно меньшем количестве оценок.
